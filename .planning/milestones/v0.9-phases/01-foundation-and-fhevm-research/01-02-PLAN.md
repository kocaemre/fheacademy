    Create a step-by-step Solidity-to-FHEVM transformation guide as a React component. This is the "Migration Mindset" teaching tool that shows how to migrate a vanilla Solidity contract to FHEVM. Structure as 6 sequential steps:

    **Step 1: Import and Configuration**
    - Left (Solidity): No special imports, standard contract
    - Right (FHEVM): `import { FHE, euint32, externalEuint32 } from "@fhevm/solidity/lib/FHE.sol";` + `contract MyContract is ZamaEthereumConfig`
    - Explanation: What the FHEVM library provides and why ZamaEthereumConfig is needed

    **Step 2: Types**
    - Left: `uint8`, `uint32`, `uint64`, `uint256`, `bool`, `address`
    - Right: `euint8`, `euint32`, `euint64`, `euint256`, `ebool`, `eaddress`
    - Left (function params): `function foo(uint32 value)`
    - Right (function params): `function foo(externalEuint32 value, bytes calldata inputProof)`
    - Explanation: Each type maps to its encrypted equivalent; function parameters need ZKPoK proof

    **Step 3: Operations**
    - Left: `a + b`, `a - b`, `a * b`, `a == b`, `a < b`, `condition ? a : b`
    - Right: `FHE.add(a, b)`, `FHE.sub(a, b)`, `FHE.mul(a, b)`, `FHE.eq(a, b)`, `FHE.lt(a, b)`, `FHE.select(condition, a, b)`
    - Left: `if (condition) { ... }`
    - Right: Cannot use `if` with encrypted bools -- use `FHE.select` instead
    - Explanation: Critical paradigm shift -- both branches of FHE.select always execute

    **Step 4: ACL Permissions**
    - Left: N/A (data is public by default)
    - Right: `FHE.allowThis(handle)` after every state mutation, `FHE.allow(handle, msg.sender)` to grant caller access
    - Explanation: Every ciphertext has an ACL; contract must grant itself access to use new ciphertexts

    **Step 5: Encrypted Inputs**
    - Left: `function foo(uint32 value) { use value directly }`
    - Right: `function foo(externalEuint32 value, bytes calldata inputProof) { euint32 v = FHE.fromExternal(value, inputProof); }`
    - Explanation: Off-chain encryption + ZKPoK proof ensures input integrity

    **Step 6: Decryption (v0.9 Self-Relaying)**
    - Left: `return value;` (plaintext)
    - Right: `return encryptedHandle;` (handle only) + `FHE.makePubliclyDecryptable(handle)` for off-chain decrypt
    - Off-chain: `@zama-fhe/relayer-sdk` `publicDecrypt()` + submit proof back
    - Explanation: Values stay encrypted on-chain; decryption is user-driven, not Oracle-driven
