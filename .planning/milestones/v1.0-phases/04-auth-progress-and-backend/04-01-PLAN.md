   - Import `ThirdwebProvider` from `thirdweb/react`
   - Create `ThirdwebProviderWrapper` that wraps children with `<ThirdwebProvider>`
   - No chain props needed — wallet connection is purely for identity

2. Create `app/api/progress/route.ts`:
   - Import supabase from `@/lib/supabase-client`
   - **GET handler:** Accept `?wallet=0x...` query param. Fetch from `progress` table by `wallet_address` (lowercased). Return `{ completions: string[] }`. If no row found (PGRST116 error code), return empty array.
   - **POST handler:** Accept `{ wallet: string, completions: string[] }` body. Upsert to `progress` table with `wallet_address` (lowercased), `completions` (JSONB), `updated_at` (ISO timestamp). On conflict on `wallet_address`, update. Return `{ ok: true }`.
   - Always `.toLowerCase()` wallet addresses to prevent case-sensitivity issues.
   - Graceful error handling: catch Supabase errors and return 500 with error message.

3. Create `components/providers/progress-provider.tsx` ("use client"):
   - React Context providing `ProgressContextValue`:
     ```typescript
     interface ProgressContextValue {
       completedItems: Set<string>;
       isComplete: (itemId: string) => boolean;
       toggleComplete: (itemId: string) => void;
       weekProgress: (weekId: number) => { completed: number; total: number };
       overallProgress: () => { completed: number; total: number };
       isLoading: boolean;
     }
     ```
   - `useProgress()` hook that throws if used outside provider
   - Storage key: `"fhe-academy-progress"`
   - **On mount (useEffect):** Read from localStorage, parse JSON array into Set. Set `isLoading = false` after read.
   - **On walletAddress change (useEffect):** When `walletAddress` becomes truthy (user connected wallet):
     1. Set a `isSyncing` ref to true to prevent Supabase writes during merge
     2. Fetch from `/api/progress?wallet={address}`
     3. Union merge: combine localStorage Set with Supabase array into new Set
     4. Update state with merged Set
     5. Write merged Set to both localStorage AND Supabase (POST)
     6. Set `isSyncing` ref to false
   - **toggleComplete(itemId):** Toggle item in Set. Write to localStorage immediately. If wallet connected AND not syncing, POST to `/api/progress`.
   - **weekProgress(weekId):** Use `getWeekItems(weekId)` from `lib/progress.ts` to get all items for that week. Count how many are in completedItems. Return `{ completed, total }`.
   - **overallProgress():** Use `getAllItems()` from `lib/progress.ts`. Count completed. Return `{ completed, total }`.
   - **isComplete(itemId):** Return `completedItems.has(itemId)`.
   - Use `useActiveAccount()` from `thirdweb/react` to get wallet address.
   - Initialize completedItems as empty Set (prevents hydration mismatch). Only populate from localStorage in useEffect (client-side only).
   - Error handling for Supabase: if fetch or POST fails, log error to console but do NOT block UI. localStorage is the resilient fallback.
